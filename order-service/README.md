# Order Service - Real-Time Payment Settlement System (RTPS)

## Overview
The **Order Service** is the central entry point of the RTPS ecosystem. Built with Spring Boot 3.2, it is responsible for managing user accounts, authenticating requests via secure JWTs, and capturing payment orders with an absolute guarantee of idempotency, financial consistency, and reliability.

---

## 🚀 Implemented Concepts (Production Hardened)

### 1. Advanced API Design & Validation
- **HTTP Header Idempotency**: Follows industry-standard API patterns (like OFSS) by extracting the `Idempotency-Key` from the HTTP Headers, keeping the JSON payload strictly focused on business logic.
- **Error Handling**: A `GlobalExceptionHandler` traps all exceptions and returns standardized JSON responses. Validation errors (via `@Valid`) explicitly state which exact field failed (e.g., `Field 'recipientBankAccount' must not be blank`) instead of throwing generic 500 errors.
- **Rate Limiting**: Protects critical endpoints like POST `/api/v1/orders` from DDoS and brute-force attacks using **Resilience4j** `@RateLimiter`.

### 2. User Management & Security
- **JWT Refresh Token Rotation**: Implemented a state-of-the-art token rotation system. Every refresh call invalidates the old token and issues a new one, mitigating replay attacks.
- **Data Encryption at Rest**: Sensitive financial fields like `recipientBankAccount` are encrypted using **AES-256** before hitting the Oracle DB, ensuring strict compliance with data protection laws.
- **Password Reset Flow**: Secure account recovery with 15-minute time-limited tokens.
- **Centralized CORS & Spring Security**: Restricted access to trusted origins and headers, managed via `WebSecurityConfig`.

### 3. Database Evolution & Oracle Integration
- **Flyway Schema Migrations**: Fully automated schema versioning (V1 through V5). Uses modern Oracle features like `GENERATED BY DEFAULT AS IDENTITY` instead of legacy sequences to avoid `Schema-validation` crashes.
- **Profile-Driven Configuration**: Uses `application-local.yml` for local development overrides, keeping `application.yml` secured with injected environment placeholders for production.

### 4. Idempotency & Reliability
- **Synchronous Kafka Transactions**: The service uses synchronous Kafka publishing within database transactions. If Kafka is unavailable, the entire transaction (Order + Idempotency Key) is rolled back instantly.
- **Idempotency Engine**: Prevents duplicate payments using a unique key per request, with an automated Spring `@Scheduled` cleanup task to purge expired keys.

### 5. Observability & Auditing
- **Lombok `@Slf4j` Logging**: High-performance, clean logging implementation using Lombok.
- **Correlation ID Tracing**: Integrated `X-Correlation-ID` across all logs and responses via `MDC` to trace requests across microservices.
- **Audit Logging**: Explicit, clean logging of security events, idempotency hits, and transaction failures for audit trails.
- **Metrics & Health**: Real-time monitoring exposed via **Spring Boot Actuator** and **Prometheus**.

## 🛠️ Custom Exceptions Map
| Exception | HTTP Status | Purpose |
| --- | --- | --- |
| `MethodArgumentNotValidException` | 400 Bad Request | Intercepted to provide exact field-level validation errors. |
| `MissingServletRequestParameterException` | 400 Bad Request | Caught to inform clients of missing URL parameters. |
| `InvalidOrderException` | 400 Bad Request | For complex business rule violations. |
| `UserNotFoundException` | 404 Not Found | For account-related lookups. |
| `Exception.class` | 500 Internal Error | Generic fallback for unhandled server crashes. |

## 📁 Project Structure
```text
com.payment.order
├── config/         # Security Config, Resilience4j, and Web Interceptors
├── controller/     # REST Controllers (Auth, Orders)
├── dto/            # Request/Response Data Transfer Objects
├── entity/         # JPA Entities (mapped to Oracle Identity columns)
├── event/          # Synchronous Kafka Producer 
├── exception/      # Custom Exceptions and Global Handler
├── repository/     # Spring Data JPA Repositories
└── service/        # Core Business Logic & Scheduled Tasks
```

## 🚀 Getting Started
1. Set active profile to `local` (or supply the required Env Vars).
2. Ensure Oracle Database is running.
3. Flyway will automatically baseline and build the required schema on startup!
